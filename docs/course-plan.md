# React 미니 블로그 강의 운영안

기초 자료를 읽은 뒤 15개 실습 단계를 순서대로 진행합니다. 실습 시간은 약 13시간 10분이며, 최종 점검 25분을 더해 5회 수업으로 운영할 수 있습니다.

## 시작 조건

- Node.js 22.13.0 이상과 Git을 설치합니다.
- `main`에서 `npm.cmd ci`, `npm.cmd run lint`, `npm.cmd run build`를 실행합니다.
- `docs/basic/README.md`의 12개 챕터를 먼저 읽습니다.
- `git status --short`의 출력이 없는 상태에서 다음 branch로 이동합니다.

Windows 11 명령과 경로는 [Windows 개발 환경 준비](./windows-11.md)를 따릅니다.

## 수업 구성

| 회차 | 단계 | 권장 시간 | 완료 결과 |
| --- | --- | ---: | --- |
| 1 | `step-1` ~ `step-3` | 165분 | JSX 화면을 정리하고 공통 컴포넌트와 route를 연결합니다. |
| 2 | `step-4` ~ `step-7` | 185분 | props, 배열 렌더링, 동적 route, 지역 state를 사용합니다. |
| 3 | `step-8` ~ `step-10` | 160분 | controlled form으로 글을 추가하고 입력 오류를 표시합니다. |
| 4 | `step-11` ~ `step-13` | 155분 | 글을 삭제·수정하고 검색 결과를 만듭니다. |
| 5 | `step-14` ~ `step-15`, 최종 점검 | 150분 | 저장 데이터를 복구하고 mock fetch와 fallback을 확인합니다. |

쉬는 시간은 수업 여건에 맞게 별도로 둡니다. 한 회차가 끝나면 `lint`, `build`, 브라우저 확인을 모두 마칩니다.

## 단계별 독립 확인

| 단계 | 수강생 확인 |
| --- | --- |
| 1 | 홈 설명 문장을 바꿨다가 원래대로 복구합니다. |
| 2 | Header 또는 Footer 문구를 props로 바꿀 위치를 찾습니다. |
| 3 | 임시 `/help` route를 추가해 연결을 확인한 뒤 삭제합니다. |
| 4 | 서로 다른 author를 전달한 카드 두 개를 비교합니다. |
| 5 | 게시글을 하나 추가하고 `key`가 중복되지 않는지 확인합니다. |
| 6 | 없는 `postId`의 안내 화면을 확인합니다. |
| 7 | 좋아요를 연속으로 눌러 이전 state 기준 갱신을 확인합니다. |
| 8 | 각 input의 state와 화면 값을 비교합니다. |
| 9 | 글을 두 개 연속 추가해 기존 글이 유지되는지 확인합니다. |
| 10 | 공백 입력과 정상 입력의 결과를 비교합니다. |
| 11 | 삭제 뒤 목록과 상세 주소의 결과를 확인합니다. |
| 12 | 수정 전후 title, content, id를 비교합니다. |
| 13 | 대문자와 소문자 검색 결과를 비교합니다. |
| 14 | 잘못된 저장값에서 초기 데이터로 복구되는지 확인합니다. |
| 15 | mock fetch 성공과 fallback을 각각 확인합니다. |

## 완료 기준

- 각 단계 문서의 실행 확인과 독립 확인을 기록합니다.
- 새 글 추가, 공백 검증, 수정, 삭제, 검색을 브라우저에서 확인합니다.
- 저장값이나 mock fetch 응답이 게시글 형식의 배열이 아닐 때 화면이 초기 데이터로 복구됩니다.
- 마지막 branch에서 `npm.cmd run lint`와 `npm.cmd run build`가 통과합니다.

막힌 경우 임의로 파일을 지우기 전에 [문제 해결](./troubleshooting.md)의 순서대로 원인을 좁힙니다.
