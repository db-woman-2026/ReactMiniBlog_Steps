# React 미니 블로그 실습 구성

Vite 프로젝트에서 JSX, 컴포넌트, 라우팅, 상태, 입력, 저장과 데이터 로딩을 구현합니다. 코드는 개인 GitHub 저장소의 `main`에 기록합니다.

## 시작 조건

- [Windows 11 개발 환경 준비](./windows-11.md)에 따라 Vite React 프로젝트를 만듭니다.
- Windows Terminal, Node.js, Git, VS Code와 GitHub CLI를 사용할 수 있는지 확인합니다.
- 개인 `react-mini-blog` 저장소의 `main`에 첫 commit이 push되어 있는지 확인합니다.
- 필요한 문법은 [React 기초 자료](./basic/README.md)에서 확인합니다.

## 단계별 결과

| 단계 | 완료 결과 |
| --- | --- |
| Step 1~3 | JSX 화면을 정리하고 컴포넌트와 route를 연결합니다. |
| Step 4~7 | props, 배열 렌더링, 동적 route와 지역 state를 사용합니다. |
| Step 8~10 | controlled form으로 글을 추가하고 입력 오류를 표시합니다. |
| Step 11~13 | 글을 삭제·수정하고 검색 결과를 만듭니다. |
| Step 14~15 | 저장 데이터, mock fetch와 fallback을 확인합니다. |

## 단계별 독립 확인

| 단계 | 확인 결과 |
| --- | --- |
| 1 | 홈 설명 문장을 바꿨다가 원래대로 복구합니다. |
| 2 | Header 또는 Footer 문구를 props로 바꿀 위치를 찾습니다. |
| 3 | 임시 `/help` route를 추가해 연결을 확인한 뒤 삭제합니다. |
| 4 | 서로 다른 author를 전달한 카드 두 개를 비교합니다. |
| 5 | 게시글을 하나 추가하고 `key`가 중복되지 않는지 확인합니다. |
| 6 | 없는 `postId`의 안내 화면을 확인합니다. |
| 7 | 좋아요를 연속으로 눌러 이전 state 기준 갱신을 확인합니다. |
| 8 | 각 input의 state와 화면 값을 비교합니다. |
| 9 | 글을 두 개 연속 추가해 기존 글이 유지되는지 확인합니다. |
| 10 | 공백 입력과 정상 입력의 결과를 비교합니다. |
| 11 | 삭제 뒤 목록과 상세 주소의 결과를 확인합니다. |
| 12 | 수정 전후 title, content, id를 비교합니다. |
| 13 | 대문자와 소문자 검색 결과를 비교합니다. |
| 14 | 잘못된 저장값에서 초기 데이터로 복구되는지 확인합니다. |
| 15 | mock fetch 성공과 fallback을 각각 확인합니다. |

## 완료 기준

- 각 단계의 실행 결과와 독립 확인을 기록했습니다.
- 새 글 추가, 공백 검증, 수정, 삭제와 검색을 브라우저에서 확인했습니다.
- 저장값이나 mock fetch 응답이 게시글 형식의 배열이 아닐 때 초기 데이터로 복구됩니다.
- 마지막 코드에서 `npm.cmd run lint`와 `npm.cmd run build`가 통과합니다.
- 개인 저장소의 `main`과 원격 `main`이 같은 commit을 가리킵니다.

막힌 경우 파일을 임의로 지우거나 다른 Step의 코드를 가져오기 전에 [문제 해결](./troubleshooting.md)에서 증상에 맞는 진단 절차를 확인합니다.
